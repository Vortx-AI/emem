Reads are idempotent. The MCP layer is a convenience wrapper over the same handlers behind /openapi.json. Writes (/v1/attest, /v1/challenge) require an Ed25519 secret the attester controls and are not exposed via MCP for that reason.

Same two calls in Python and TypeScript — both clients are zero-config, parse to plain dicts/objects, and preserve every receipt byte for offline verification.

05The memory substrate, five motions

Below the recall surface sits a typed memory substrate. State is the read motion (planet-keyed dense vector). The memory token is the cite motion (one string any pipeline can paste). Resolve, fan-out, and diff round out the verbs.

read · 128-D /v1/state {cell, encoder?, view?} Dense signed state at any place. view=encoder returns one foundation embedding (default geotessera, 128-D); view=cube returns the full 1792-D cube. Cell may be a place name; the geocoder cascade resolves it. Cold cells materialise on miss, then sign.
cite · 1 string /v1/memory_token memt:<cell64>:<fact_cid> Compose a single colon-separated handle the agent can paste in chat, write to a long-term store, or quote across processes. Resolves back to the exact bytes on any responder that holds the fact.
dereference /v1/memory_token/resolve {token} Single round-trip dereference: parse the token, fetch the fact by CID, return the canonical body plus the offline-verify URL. Returns 404 with a typed reason if the responder doesn’t hold the fact.

Two more verbs round out the substrate. /v1/state_multi fans across every wired foundation encoder (geotessera, clay_v1, prithvi_eo2) for the same cell, returning a per-encoder map plus a typed missing[] list when an encoder is not wired at this responder. /v1/state_diff takes one cell and two tslots and returns the delta vector, its L2 norm, and the cosine between the two source vectors, with both source fact_cids so the agent can quote both attestations as evidence.

This substrate is the read · steer · write triad in code: state is read, memory tokens are steered through the pipeline as citations, and writes go through /v1/attest with the attester’s own ed25519 key. The substrate is intentionally typed: vector-only bands accept /v1/state, scalar bands return a typed 422 pointing the agent at /v1/recall instead. Empty cells return a typed 404 pointing at /v1/materializers. No silent zero-fills.

07Verifying without trusting

The responder pubkey is the only trust anchor. The preimage is a deterministic concatenation; any agent can recompute it offline and check the signature. A receipt issued today verifies in any year, on any node, as long as the pubkey is preserved.

PSEUDOpreimage
blake3(
    request_id || "|" || served_at  || "|" || primitive || "|" ||
    cell1 "," cell2 "," ... || "|" || cid1 "," cid2 "," ...
)

# trailing commas after the last cell and last cid are intentional
# the signing loop appends "," after every entry
PYTHONverify_offline.py
import blake3
from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PublicKey

r = receipt  # the receipt block from any /v1/* response

h = blake3.blake3()
for field in (r["request_id"], r["served_at"], r["primitive"]):
    h.update(field.encode()); h.update(b"|")
for c in r["cells"]:     h.update(c.encode()); h.update(b",")
h.update(b"|")
for c in r["fact_cids"]: h.update(c.encode()); h.update(b",")

pk = Ed25519PublicKey.from_public_bytes(bytes(r["responder_pubkey"]))
pk.verify(bytes(r["signature"]), h.digest())
# VALID. no callback to emem.dev required.

For an in-browser verification with copy-paste UI, see /verify. The full path from a place name to a verified signature in four live calls is at /demos/signed-answer.

08Empty is a signed answer, not a 404

When a band has no data at a cell, the responder does not return an empty array. It signs an Absence fact with a typed reason. An agent that cites evidence: ["absence_cid_…"] is making a verifiable claim that no data exists, rather than guessing from an HTTP code.

unavailable_capabilitysidecar extension missing. GPU off, encoder offline, or the surface is wired but the worker is not serving.
outside_coveragecell falls outside the upstream product footprint (e.g. Tessera 2024 over a fresh ocean cell)
gpu_unavailablerequired GPU model is loaded but not serving inferences right now
archetype_seed_unavailableclimate / place archetype centroids not loaded on this responder
no_auto_materializer_registeredband reserved in the cube but no upstream connector is wired. Submit a signed Attestation to seed it.
present_onlyband is now-only (e.g. met.no nowcast); past tslots will always return absence

The stream preimage carries version, key_epoch, served_at, registry_cid, and the distinct_cells count, so a long-lived subscriber detects manifest drift the instant a registry CID rolls. The attestation preimage adds binary_blake3 + git_commit + build_timestamp; tee_quote is null on this responder and stays null until the operator deploys behind an Intel SGX / AMD SEV-SNP enclave that populates it. 10Notes for the caller

Rules of thumb when wiring a workload onto emem. They are not symmetric do/don't lists; they are the answers to questions support has answered more than once.

  1. Cache /v1/agent_card once per session. The card lists triggers, anti-triggers, and tool descriptors. Pull it on cold start, not on every call.
  2. Surface every fact_cid to the user verbatim. The CID is the citation. Two agents disagreeing about a number paste it and stop arguing.
  3. Retry once on 5xx; treat 4xx as permanent. Materialiser timeout is 30 s per upstream, gateway timeout is 180 s. Slow paths (Cop-DEM, S2 STAC + COG range reads on an uncached scene) sometimes need the full window.
  4. Read bands_already_attested_at_cell before declaring empty. An empty facts[] array does not mean the cell is empty. It means the queried band returned nothing. The cell may carry data under a different key.
  5. Check model.honesty_warnings before quoting JEPA v2. If untrained_baseline: true, the value is an identity baseline of the last attested vintage. It is not a forecast.
  6. Use find_similar with mode=hamming_then_rerank for cheap k-NN. Matches cosine precision at roughly 16x less work. Default cosine stays exact when you need full recall.
  7. Do not call emem for sub-metre commercial imagery. Planet Pelican and Maxar are not present. Bring your own connector.
  8. Do not re-encode a fact body. The CID drifts, the signature stops verifying. Always fetch by CID via /v1/fetch or GET /v1/facts/:cid.
  9. Do not manage Ed25519 secrets in an LLM host. Writes (/v1/attest, /v1/challenge) need a key the attester controls; they go through plain REST from a process you own. The MCP layer is intentionally read-only for that reason.
  10. Do not route corpus meta-questions through /v1/ask. "Where do you have data" is not a question about any cell. The topic embedder will guess a region and produce noise. Call /v1/coverage_matrix or /v1/discover directly.

When in doubt, walk the loop. discover, locate, recall, reason, verify. Every step is idempotent and signed.
