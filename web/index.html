emem

Earth memory for AI agents.

Every place on Earth gets a stable address called a cell64, about 10 m on a side. Facts about that place live at the address: elevation, weather, land cover, foundation embeddings, whatever a materializer has signed and put there. The algebra is Cell × Band × Tslot → Fact. A cell is the where, a band is the what (one of 34 named bands at /v1/bands), a tslot is the when. Every read comes back with an ed25519 signature and a blake3 CID over the bytes you got. Paste a CID into Slack and someone else can pull the same fact and verify it themselves. No API keys. No accounts.

Point an MCP client (Claude, Cursor, Cline, Gemini, ChatGPT) at https://emem.dev/mcp and the tools show up. Or hit plain HTTP. GET /v1/locate?place=Mount+Fuji returns the cell. GET /v1/elevation?lat=35.36&lng=138.73 returns 3709 m with its signature attached. For agents bootstrapping the surface, /v1/discover fits in 970 bytes, /llms.txt and /agents.md describe the protocol in prose, /openapi.json has the full schema, the spec is at /spec.md. Run a node yourself; the receipts are still verifiable when we're not.
